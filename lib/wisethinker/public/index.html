A Book Review

The Docker Book: Containerization is the new virtualization

Written by James Turnbull

Review by Dave Vallance

Nov 20, 2014

The Quick and Dirty Summary

There is a lot of good documentation about docker on the official site and through out the web, so this book is NOT a must read, however it does present the information in a nice flow and I found it an enjoyable read. I walked away feeling I had a great overall understanding of Docker itself, and a better focus on how I wanted to start using it!

Introduction

I didn't know much about Docker, other then the basic idea of what it tries to accomplish. That being the ability to build an application, possibly of high complexity and with lots of moving parts, and then package it up in a standardized container (akin to a shipping container). Once in a shipping container it therefore should be extreamly easy (in therory) to move this tested and working application around without worrying about any inconsistentcies amoungst the different running environments. Whether on another developers laptop, in a QA environment, or out to production servers; you application should function consistently.

This concept of building an application and then shipping it off, knowing it will "just work", is what got me excited about Docker and brought me to read this book.

How does it read? What does it cover?

I found the book reads very well. Theres your standard into information, such as who the book is written for, acknoledgments, and blurbs on numerous individuals that helped develop it. Then after giving some quick info on what container virtualization is and how its different then hypervisor virtualization, the book gets into Docker itself.

It gives some good detail on what Docker is, what it does, and how you can make use of it. At this point, if your like me, you'll be jazzed up to start trying it out and the book flows nicely right into installation. It lets you know the requirements requirements and gives step by step examples of installation on 4 different systems Ubuntu, Red Hat, OS X and Microsot Windows.

I always go to the source(https://docs.docker.com/installation/#installation) to make sure I have the most recent installation information (An this book was spot on for an Ubuntu install).
